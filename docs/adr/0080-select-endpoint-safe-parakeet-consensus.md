# ADR-0080: Select endpoint-safe Parakeet consensus

Date: 2026-07-17
Status: accepted

## Decision
Extend ADR-0079's local exact-acoustic verifier with sherpa-onnx `nemo_transducer` offline finals and select the checksum-pinned English Parakeet Unified 0.6B int8 export plus pinned CUDA Faster-Whisper Small for this machine. Keep the committed SenseVoice profile unchanged and expose the selection only through the existing installer and `./live.sh` entry (`--final-asr parakeet-unified-en --final-verifier faster-whisper-small`). Non-empty text may still change only on exact independent acoustic quorum. Add one bounded absence rule: two successfully decoded empty results may suppress an established final only when that final has exactly one normalized word, is the streaming rendering, and has no STOP, confirm/deny, or mode-switch meaning. Add one bounded control exception for the NeMo path: a retained owner-attested transcript pair may cross the ordinary control-change fence only when Parakeet and Faster-Whisper have identical complete words and controller meaning and owned speech duration is inside the retained evidence window. Errors, unavailable/skipped models, one empty result, multi-word finals, unmatched controls, ties, and no quorum retain the established final. Generic verifier changes keep losing owner and device-action authority.

## Context / why
The production-path six-recording replay, not a whole-file decode, is the selection gate. The ADR-0079 SenseVoice/Small profile measured selected WER 0.04 (5/6 exact, one insertion). Parakeet TDT v3 was 0.00 on whole recordings but tied production at WER 0.04 after endpoint segmentation, so it is not selected. The newer English Parakeet Unified export was 0.00 offline on the endpoint path, but one spurious one-word streaming endpoint survived because both stronger models returned empty and ADR-0079 treated empty as no evidence. The bounded double-empty veto removed that insertion: selected WER 0.00, 6/6 exact, one win, zero losses, six exact consensuses plus one empty veto. Three repeated direct decodes were deterministic; direct Parakeet and Small each reached 0.00 and retained the labelled STOP. Parakeet CPU warm p50 was about 117 ms and Small GPU warm p50 about 35 ms. Small.en and Base each regressed STOP; span repair added no gain over exact selection. The strict recorded gate initially exposed the historical garbled-STOP case; transcript-free inspection proved both independent models assigned the same STOP class, so the existing exact pair/duration attestation is retained only behind their two-model agreement. A final red-team seam probe found that the first draft let NeMo text pass through the legacy single-model agreement guard before quorum; the selected implementation instead keeps NeMo evidence-only on disagreement, empty, error, and verifier unavailability, and strips action trust on every agreed rewrite.

## Consequences
`SherpaConfig`, readiness, replay provenance, setup, and the single app/launcher entry understand the four-file NeMo transducer contract with sherpa-onnx's documented `feature_dim=80` constructor setting. The export records an internal 128-wide encoder input; a transcript-free A/B decoded all six clips exactly with either constructor value, so the documented runtime contract is retained rather than inferred from the ONNX tensor shape. Setup verifies archive SHA-256 `99f63605b3a85a54c250c0869670a687b7d6598a47bf2421515e1f839a76e150` plus the four extracted-file hashes, rejects a mismatched cached directory, stages atomically, and publishes config only after completion. The selected runtime pins sherpa-onnx 1.13.3; readiness performs a discarded NeMo decode before microphone startup, while aggregate replay fails if either selected offline model or verifier errors or never completes a decode. Aggregate reports bind the joiner and complete decoder/thread contract. The strict recorded suite remains 9/9 and the protected STOP path requires two exact acoustic votes. The development corpus is small and was used during selection, so this is machine-local development evidence, not a universal default or calibrated confidence claim. A disjoint held-out corpus and a fresh physical `./live.sh` microphone/speaker A/B remain required; no live hardware result is claimed. This ADR supersedes ADR-0079 and still refines ADR-0078.
